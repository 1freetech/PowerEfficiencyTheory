Power Efficiency Theory proposes that the value of a computational system scales with its computational energy productivity, defined as the amount of computation produced per unit of energy. In Bitcoin mining this relationship can be expressed through the interaction between computational power (hashrate) and energy efficiency (Joules per Terahash). If computational power increases while efficiency remains constant, the system produces more work and its productive capacity rises. If energy efficiency improves while computational power remains constant, the same computational work can be produced using less energy, which also increases productivity. Because either improvement increases the ratio of computation to energy input, the intrinsic productive value of the machine or protocol increases as this ratio compounds over time. This Python model explores how these interacting variables evolve over time and how their compounding relationship may influence the structural value of computational systems such as Bitcoin mining hardware and the Bitcoin network itself.
