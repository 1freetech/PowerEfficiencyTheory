# Network Regime Model

## Purpose
This file outlines the next simulation upgrade: moving from machine and theory curves into a fuller Bitcoin mining network regime model.

## Why this matters
Power Efficiency Theory becomes more commercially useful when it can speak not only about isolated machine efficiency, but also about broader mining conditions.

## Network-level variables to add
- network difficulty trend
- block subsidy regime
- fee regime assumptions
- hardware generation turnover
- fleet retirement pressure
- energy cost dispersion
- uptime / curtailment assumptions

## Why this is the right next layer
The repo already has:
- deterministic theory curves
- machine-derived efficiency modeling
- probabilistic simulation outputs

The next serious improvement is to connect those ideas to changing Bitcoin network conditions.

## Practical outputs this could support
- miner resilience scenarios
- efficiency-vs-difficulty stress testing
- premium research reports
- sponsor-backed analytical explainers
- dashboard views for mining cycles

## Minimal simulation concept
A future file could model yearly changes in:
1. machine efficiency
2. network difficulty
3. effective revenue pressure
4. retained productivity advantage
5. margin compression or expansion

## Commercial relevance
This is one of the strongest ways to move the repo from theory toward a productized mining-analysis engine.
